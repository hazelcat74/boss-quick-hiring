# Boss Daily Brief 工作流

## 功能概览

一条指令，完成今天 Boss直聘 的全部招聘动作：

- 拉取候选人列表 - 获取今天的 Boss直聘 候选人
- 批量获取简历 - 自动获取候选人简历信息
- AI 汇总分析 - 按职位分组整理为 Markdown 格式
- 创建飞书云文档 - 将分析结果自动创建为飞书文档
- AI 初筛评级 - 对候选人进行智能评分和筛选
- 初筛匹配度报告 - 生成详细的简历匹配度评估报告
- 自动发索要简历消息 - 对合适的候选人发送消息
- 操作记录归档 - 将结果记录到飞书文档中

## 触发方式

在任意 AI 助手中说以下指令即可自动触发：

- 「帮我处理今天的 Boss直聘」
- 「做今天的简历 brief」
- 「拉一下直聘候选人」

## 工作流程

1. **执行 opencli boss chatlist** 获取候选人列表
2. **顺序获取简历**（使用 --verbose 模式，逐个执行）
3. **AI 整理 Markdown**（按职位分组）
4. **创建飞书云文档** 存储分析结果
5. **AI 初筛打分** 对每个候选人进行评级
6. **生成初筛匹配度报告** 详细分析简历与职位的匹配情况
7. **操作记录追加** 到飞书文档（优先约谈名单 + 不合适列表）

## 初筛匹配度报告

### 评估模型

采用 "**硬性门槛 + 核心匹配 + 背景加分 + 风险修正**" 四段式模型：

| 模块 | 权重 | 评估内容 | 说明 |
|------|------|----------|------|
| 硬性门槛 | 是否通过 | 学历下限、年限下限、岗位必须经验、排除项 | 不通过则直接淘汰 |
| 核心匹配度 | 50% | 岗位核心能力、关键项目经历、相关行业/场景经验、技能关键词命中率 | 决定是否值得优先推进 |
| 背景竞争力 | 30% | 学历与学校、公司平台、职级水平、项目复杂度、成果影响力 | 用于区分"合格"和"优秀" |
| 稳定性与风险 | 20% | 在职时长、跳槽频率、经历连续性、信息完整度、潜在风险项 | 避免高分候选人存在明显隐患 |

### 详细评估标准

#### 1. 硬性门槛
- **最低学历要求**（如本科及以上）
- **最低工作年限要求**（如 3 年以上）
- **必须具备的岗位经验**（如 SoC 集成、AI Infra、搜索推荐、招聘全流程等）
- **必须具备的关键技能**（如 Python、Verilog、UVM、C++、渠道开源能力等）
- **排除项**（如纯转行、长期空窗无法解释、频繁短跳、完全不相关赛道）

#### 2. 核心匹配度（50%）
- **岗位能力匹配**（20分）：是否具备 JD 中最关键的 2~3 项核心能力
- **项目/场景匹配**（15分）：是否做过与目标岗位相似的项目、业务场景或行业场景
- **技能关键词匹配**（10分）：是否命中岗位关键技能、工具、方法论或协议栈
- **成果证明**（5分）：是否有清晰结果，如负责核心模块、主导项目、完成交付、拿到业务结果

**评分逻辑**：
- 强匹配：45~50分
- 中匹配：30~44分
- 弱匹配：15~29分
- 明显不匹配：0~14分

#### 3. 背景竞争力（30%）
- **学历与学校**（10分）：不建议只看最高学历，建议同时结合学校层级与专业相关性
- **公司平台**（8分）：是否来自目标行业头部公司、标杆团队或高要求环境
- **职级与职责**（6分）：是否承担过更高复杂度职责，而不只看 title 本身
- **项目复杂度/成果影响**（6分）：是否参与过高复杂度项目、关键系统或高影响力业务

**建议原则**：
- 背景强是加分项，但不能替代岗位匹配
- 对于强执行岗或紧缺岗，应降低学历光环权重、提高实战经验权重

#### 4. 稳定性与风险修正（20%）
- **稳定性**（8分）：近 3~5 年平均在职时长是否健康
- **经历连续性**（4分）：是否存在频繁断档、跨度过大的经历跳跃
- **动机合理性**（4分）：离职原因、求职动机是否可解释
- **信息完整度与真实性**（4分）：简历是否清晰、逻辑是否自洽、是否存在明显夸大

**扣分项**：
- 每段经历普遍低于 1 年，可扣分
- 关键经历描述模糊、无法判断产出，可扣分
- 岗位方向频繁切换、路径混乱，可扣分

### 评级结果输出

AI 输出以下结构化结果：
- **是否通过硬性门槛**：通过 / 不通过
- **总分**：100 分制
- **核心匹配度得分**：XX / 50
- **背景竞争力得分**：XX / 30
- **稳定性与风险得分**：XX / 20
- **推荐结论**：优先推荐 / 推荐 / 待定 / 不推荐
- **关键理由**：3 条以内，说明加分点和风险点

### 星级评分对应关系

| 星级 | 评分范围 | 推荐结论 | 自动动作 |
|------|----------|----------|----------|
| ⭐⭐⭐⭐⭐ | 85分及以上 | 优先推荐 | 优先约谈 |
| ⭐⭐⭐⭐ | 70~84分 | 推荐 | 优先约谈 |
| ⭐⭐⭐ | 60~69分 | 待定 | 可选择约谈 |
| ⭐⭐ | 40~59分 | 不推荐 | 不操作 |
| ⭐ | 20~39分 | 不推荐 | 操作记录中列出 |
| ❌ | 0~19分 | 排除 | 操作记录中列出 |

### 报告用途

- 帮助招聘方快速了解候选人与职位的匹配情况
- 为面试决策提供数据支持
- 识别候选人的核心优势和潜在风险
- 提高招聘效率和准确性

## 常见问题

- **Q: opencli: command not found**
  **A: 运行 `npm install -g @jackwener/opencli` 安装。**

- **Q: Browser Extension is not connected**
  **A: 在 Chrome 中安装 opencli Browser Bridge 扩展并确认已连接。**

- **Q: Cookie 已过期**
  **A: 在 Chrome 中重新登录 Boss直聘招聘端（ `https://www.zhipin.com`），然后继续。**

- **Q: 飞书报 missing_scope 权限错误**
  **A: 重新运行首次使用配置中的一次性授权命令。**

- **Q: resume 命令触发页面跳转或报 Cookie 已过期**
  **A: 必须使用 --verbose 标志。不加时 opencli 会走触发页面跳转的路径导致 Cookie 失效。**

## 注意事项

- **权限要求**：需要飞书文档的创建和写入权限
- **网络连接**：需要稳定的网络连接以获取候选人信息
- **Chrome 登录**：必须在 Chrome 中登录 Boss直聘招聘端
- **Cookie 有效性**：如果 Cookie 过期，需要重新登录 Boss直聘
